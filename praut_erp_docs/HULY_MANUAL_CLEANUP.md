## Known Future Cleanup Categories

Track these during Huly setup:

- Accidentally created normal Cards instances instead of `Settings -> TYPES`.
- Duplicate document imports.
- Missing required fields.
- Missing required views.
- Broken references or associations.
- Wrong owners or missing owners.
- Public or overly broad visibility on sensitive spaces.
- Expired or overbroad invite links.
- Guest/anonymous access enabled unintentionally.
- Import tool limitations requiring manual field type adjustment.
- Automation rules created with actions stronger than alert-only.
- AI features set to `povoleno` where `povoleno se schvalenim` or `zakazano` is required.

## Cleanup Safety Rules

- Do not delete Huly objects automatically unless explicitly approved.
- Record questionable duplicates first.
- For permissions and external sharing, ask before changing live systems.
- For production data, back up before cleanup.
- For generated/imported content, keep source paths and object names in evidence.
